## 7. 포인터

- 주소 : 8byte

- `int *p = &a;`

    - `&p` 에 `&a`가 존재한다.

    - `*p` : 주소를 저장하고 있으면 사용.

    - 즉, `&p`에 `&a`가 존재해서 `*p`가 된다.

    - `int` 포인터 : 크기와 특징을 알 수 있으니까 => 포인터에 타입이 있는 이유

    - 사실 주소의 크기는 모두 동일 (8byte)

    - void pointer : 블랙홀 같은 놈. (여러 것들을 담아서 처리하고자 해서 존재하는 것)

### 이중 포인터 - 살짝 이해가 안됨.

- `int **pp = &p;`

    - `*pp` : 주소가 들어갈 것이기 때문에 8byte

## 8. 배열과 포인터의 관계

```
int arr[3] = {10, 20, 30};
int *p = arr;
```

- `&arr` : 3칸 잡아놓고 가리키는 것은 0번 인덱스를 가리킴

- 즉, arr == &arr[0] == p

- `&p` : `arr` 대입

- Why? 배열은 첫 칸을 가리키기 때문에 `int *p = arr;` 부분은 `&arr`과 비슷한 의미라고 생각하자.

- arr은 배열이기 때문에 인덱싱이 가능하지만 포인터로 가면 인덱싱 불가능.

- 포인터는 연산만 가능. 

    - `*(p+1)` : `+1`은 size 만큼 하기 때문에 `arr[0]`이 size가 100이고 `arr[1]`이 size가 104이면 `*(p+1)`은 101을 하는 것이 아닌 size만큼해서 104의 값를 가리킨다.

    - 즉, `arr[1]` = `*(p+1)`

    - 쓰고 나서 free를 해줘야 함.

## 11. struct와 포인터를 같이 쓰기 - 길 잃음...

- padding이 struct가 배열일 때를 의미하는 건지, 일반적으로 그러는건지 잘 모르겠음.

- 메모리 효율은 크기가 큰 것 -> 작은 것으로 하는 것이 좋음.

- int이면 4byte이므로 4의 배수에서 실행하므로

```

int id;
```

- 위와 같은 경우는 100이 char은 1byte이고 3byte가 padding된 후, int가 104부터 실행
